Typography
Two typefaces. Poller One for display and bold accent moments. Roboto for everything functional. Body text is 14px. Sizes and weights are intentional — don't improvise.

Agent Entry
The entry affordance is the agent's brand anchor: a dark square with a pixel-rendered OpenCreator diamond. It should feel tactile, slightly mysterious, and richer in motion than the rest of the system without becoming ornamental.
Pixel Identity
Matrix construction

A 15x15 pixel-dot matrix uses 4px blocks with 1.5px gaps. The logo sits inside a 48px square with 14px corner radius.

Always-dark anchor

The base button fill stays #0D0D0D regardless of page theme so the entry reads as a consistent agent object, not another canvas control.

Ambient idle motion

Idle state uses a radial brightness wave plus sparse firefly sparkles. The effect should read as quiet energy, not shimmer.

Hover behavior

Hover introduces two things: magnetic pixel pull toward cursor and neon bleed from the four modality colors. This is the only place brand glow is allowed to bloom this much.

Click behavior

Click compresses the button and emits an expanding ring of 30 tiny particles. Keep it short, square, and precise.

Interaction contract

Scale transitions use --ease-spring. Hover to 1.06, active to 0.96, never beyond those bounds.

Thinking States
Thinking UI should feel legible and alive without stealing focus from the conversation. Pixel icons stay small, text stays quiet, and completion states turn into compact collapsible summaries rather than loud banners.
Thinking Inline
Use the inverted mini diamond here: dark pixels on a light background. The icon breathes; the label waves by opacity and only lifts 1px at peak.
Thought Complete
Thought for 16 seconds
Reasoned through entry affordance hierarchy, mapped tool-state semantics, and selected low-amplitude motion rules before any tool UI was revealed.
Completed thinking states stay compact by default. Use them as a disclosure header, not a transcript dump: one sentence summary, duration label, and a quiet chevron.
Controls & States
The left-side agent feed already has a usable control grammar. This section consolidates the real button families, the shared interaction ladder, and the execution signals so future UI changes stop inventing one-off treatments.
Control Families
Icon Button
Used for header actions, attachment, and lightweight feedback. Base is transparent-to-soft fill; hover strengthens contrast without adding heavy depth.
Glass Pill
Used for capability pickers like asset and skill entry points. Keep the shell translucent, neutral, and slightly lifted only on hover.
Primary Send
The main composer commit action. Idle stays dark and compact; hover can invert into the brand green state, but the footprint and geometry must stay identical.
Pain-point Flip Hook Lead with the “easy to smudge” claim, then resolve with proof.
Suggestion Row
Decision trays use a quiet bordered row, not a floating card. Selection settles with grayscale confirmation and a slight dim, while non-chosen options fade harder.

This shell stays narrow and informational. Status lives in the left indicator and the meta row; the container itself should not shout.

Interaction State Ladder
Rest
Neutral surface, readable copy
Default state is flat and quiet: white surface, thin border, no semantic color.
Hover
Slight contrast increase
Hover only raises contrast and surface density. Avoid deep shadows or hue jumps for ordinary controls.
Pressed
Compressed and tactile
Pressed feedback is mostly geometric: a tiny scale compression and a denser surface, not a new visual language.
Chosen
Settled selection
After selection, keep the chosen row slightly dimmed instead of brighter. It should read “resolved”, not “still asking for attention”.
Inactive
Non-chosen options fade first
When a decision is locked, unselected siblings lose emphasis by opacity and contrast, while layout and spacing remain unchanged.
Disabled
No action available
Disabled means unavailable, not loading. Use lighter grayscale, remove hover response, and keep motion out of the shell.
Status Signals
Thinking

Use the compact diamond language for reasoning. Motion stays inside the icon and should never animate the entire container.

Tool Running

Tool execution uses the thin ring spinner already present in the queue. Keep it grayscale so execution can coexist with any content type.

Done

Completion is a dark ring, not a filled dot. It should read final and quiet, especially when stacked in a to-do list.

Pending

Pending is the lightest ring in the system. It marks future steps without pulling attention away from the current task.

Acknowledged

Feedback and confirmation states settle into stronger neutral fill, not colorful badges. The signal is “recorded”, not “celebrated”.

Feed Rhythm Rules
Sender-change gap: 24px

Every time authorship switches (user to agent, agent to user), insert a space-6 gap. This is the primary visual separator in the conversation.

Tool cascade row gap: 4px

Tool pills within the same group stack with only space-1 between them. This keeps the cascade visually tight and scannable.

Tool-to-response gap: 16px

Between a tool cascade block and the agent's text response that follows, use space-4. Enough separation to be distinct, not enough to break the reading flow.

User bubble: radius-lg, clipped corner

Right-aligned, 12px 16px padding. Bottom-right corner uses radius-xs to mark authorship. Shadow-1 only, not elevated.

Agent text: no wrapper card

Agent responses render as plain body text (14px / 1.65 line-height) without a surrounding card. The text itself is the surface, keeping the feed light.

Input bar: shadow-2, focus to shadow-3

The input starts at subtle elevation and lifts on focus. Radius lg, padding 12px 16px. Send button uses ink background.
